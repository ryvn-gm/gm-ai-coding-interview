# 歡迎來到 Signal / Algorithm Engineer 面試！

> 🌐 **語言：** 繁體中文｜[English version](welcome.en.md)

Hi，很高興你來到這一關！

這封信是幫你提前了解面試流程和需要準備的東西，讓你當天可以專注在解題上。

## 面試概覽

| 項目 | 說明 |
|------|------|
| 形式 | Live Coding + 口頭設計 |
| 總時間 | 約 90 分鐘 |
| 語言 | Python |

## 流程

面試分成三個部分，中間不會休息但節奏不會太趕：

1. **Task 1（約 30 分鐘）** — 在一個現有的 Python codebase 上加入新功能
2. **Task 2（約 40 分鐘）** — 分析一份資料並驗證產品結論
3. **Task 3（約 15 分鐘）** — 口頭系統設計討論，不用寫 code

每個 task 結束後會有幾分鐘的問答時間。

## 需要準備的東西

- **你自己的電腦**，可以跑 Python 的環境就好
- **Python 套件**：請先確認有安裝 `numpy`、`scipy`、`pandas`（如果沒有，面試前裝好就行）
- **你習慣用的 editor / IDE**，用什麼都可以

## 關於 AI 工具

**全程開放使用任何 AI 工具，沒有限制。**

ChatGPT、Claude、Copilot、Cursor，想用什麼都可以。我們在意的不是你有沒有用 AI。

## 面試時的三個小提醒

這不是評分項目，但會幫助我們更了解你怎麼想事情，也讓你的表現更容易被看到。

### 1. 把思考說出來（think out loud）

我們很重視你的**思考過程**，不只是最終結果。實作的時候請邊做邊講，例如：

- 「我現在打算先讀一下 code 看架構」
- 「我覺得這裡有個問題，因為⋯⋯」
- 「我要問 AI 一個關於 X 的問題，因為我想確認 Y」
- 「我現在卡在 Z，想試試看 W 這個方向」

如果你安靜地寫 code 寫了十幾分鐘，我們會看不到你的思考路徑，這對你不利。**真的想不到要說什麼也可以說「我在想 XX」。**

### 2. 遇到不清楚的題目，自己定義 scope

題目可能會有模糊的地方，這是**故意的** — 我們想看你怎麼處理。在真實工作中，需求本來就會有不清楚的地方，我們希望看到的反應是：

1. **先說出你看到的不清楚點**：「我覺得 X 這個地方有點不確定」
2. **自己做合理的假設**：「我先假設 X 是 Y 的情況」
3. **在這個假設下繼續推進**：「基於這個假設，我打算這樣做⋯⋯」

**不要只說「題目不清楚」就停下來。** 你可以問我們問題（我們會盡量回答），但更好的做法是先做合理假設、講出來、然後動手。我們完全接受你做的假設跟我們原本想的不一樣，重點是你能說明為什麼。

### 3. 不會的東西，請主動查或問 AI

不知道某個函式怎麼用？不確定某個概念？**請直接上網查或問 AI**，這完全沒問題、不會扣分。

盲猜反而會讓我們不知道你是真的不會還是只是沒查清楚。「不會就查」是我們很看重的工程師習慣。

## 最後

如果有任何問題，隨時問我們。面試當天也一樣 — 有任何不清楚的地方直接問，這不會扣分。

期待跟你聊！
