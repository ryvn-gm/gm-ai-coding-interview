# Task 3 — 口頭系統設計

> 🌐 **語言：** 繁體中文｜[English version](README.en.md)

這個任務是口頭進行的，不需要寫程式。

## 任務說明

我們的戒指 SDK 要整合到第三方健康 app。對方工程師說，他們希望每次偵測到一個 rep，就透過 webhook 即時推送到他們的 server。請你設計這個整合方案。

## 形式

- 口頭討論，不開電腦
- 可以在紙上或白板上畫圖

## 面試時的小提醒

詳細說明請見根目錄的 `welcome.md`，這裡是快速版：

- **邊想邊講**：這是口頭討論，請把你的思考路徑、疑問和假設都說出來。沉默思考三分鐘對你不利。
- **遇到題目不清楚的地方**：先講出你看到的模糊點，然後自己做合理假設並說明。例如「我先假設第三方需要的是『不漏資料』而非『秒級即時』，基於這個假設我會這樣設計⋯⋯」。不要等我們把所有細節都告訴你。
- **不會的東西**：你可以說「我對 X 不熟，可以查一下嗎」，沒問題。

祝順利！
